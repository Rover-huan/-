# SmartAnalyst 每日本地运行手册

这份文档适合每天重新打开电脑后照着操作。目标是启动本地 0 成本体验版，不涉及正式服务器部署。

## 一、每天重新打开电脑后的准备工作

1. 打开 Docker Desktop。

   等 Docker Desktop 显示已经启动完成，再继续下面命令。

2. 打开 VS Code。

3. 打开 PowerShell 终端，进入项目根目录：

```powershell
cd C:\Users\lenovo\Desktop\实验
```

4. 确认 `.env.local-demo` 存在：

```powershell
Test-Path .env.local-demo
```

如果返回 `True`，继续。

如果返回 `False`，说明本地 demo 配置文件不存在。可以先从模板复制一份：

```powershell
Copy-Item .env.local-demo.example .env.local-demo
```

然后在本机编辑 `.env.local-demo`，填入本地 demo 需要的配置。不要把里面的 API Key、`SECRET_KEY`、管理员密码发到聊天里，也不要提交到 GitHub。

5. 启动本地 demo：

```powershell
docker compose -f docker-compose.local-demo.yml --env-file .env.local-demo up -d
```

## 二、如果改过代码或依赖，需要重建

如果刚改过 Python/前端代码、Dockerfile、依赖文件，或者不确定镜像是不是旧的，用这个命令：

```powershell
docker compose -f docker-compose.local-demo.yml --env-file .env.local-demo up -d --build --force-recreate
```

这个命令会重建镜像并重新创建容器，但不会清空 Docker volume 数据。

不要随便执行：

```powershell
docker compose -f docker-compose.local-demo.yml down -v
```

`down -v` 会清理 demo 数据，只有明确想清空数据时才使用。

## 三、检查服务是否启动成功

```powershell
docker compose -f docker-compose.local-demo.yml ps
```

正常情况下应该看到这些服务：

- `redis`
- `api`
- `worker-analysis`
- `worker-render`
- `beat`
- `nginx`

重点看：

- `api` 是 `healthy`
- `redis` 是 `healthy`
- `nginx` 显示 `0.0.0.0:8080->80`
- 两个 worker 都是 `Up`

## 四、查看日志

查看实时日志：

```powershell
docker compose -f docker-compose.local-demo.yml logs -f
```

只看最近日志：

```powershell
docker compose -f docker-compose.local-demo.yml logs --tail=100
```

只看某个服务日志：

```powershell
docker compose -f docker-compose.local-demo.yml logs -f api
docker compose -f docker-compose.local-demo.yml logs -f worker-analysis
docker compose -f docker-compose.local-demo.yml logs -f worker-render
docker compose -f docker-compose.local-demo.yml logs -f nginx
```

退出日志：

```text
Ctrl + C
```

`Ctrl + C` 只退出日志查看，不会停止服务。

## 七、日常使用流程

1. 打开 `http://localhost:8080`。
2. 注册或登录账号。
3. 上传小文件，建议只上传测试数据，不要上传隐私数据。
4. 等待分析生成候选图。
5. 勾选候选图。
6. 生成报告。
7. 下载 `DOCX`、`IPYNB`、`TXT`、`ZIP`。

本地体验版建议：

- 一次只跑一个任务。
- 文件尽量小。
- 不要多人同时重度测试。
- PDF 默认关闭，不要开启。
- Word COM 自动目录默认关闭，不要开启。

## 八、停止本地服务

停止容器，但保留数据：

```powershell
docker compose -f docker-compose.local-demo.yml down
```

再次启动时执行：

```powershell
docker compose -f docker-compose.local-demo.yml --env-file .env.local-demo up -d
```

## 九、常见问题

### 1. Docker Desktop 没启动怎么办

现象：

- `docker compose` 报连接 Docker daemon 失败。
- Docker 命令长时间无响应。

处理：

1. 打开 Docker Desktop。
2. 等待 Docker Desktop 完全启动。
3. 重新执行：

```powershell
docker compose -f docker-compose.local-demo.yml --env-file .env.local-demo up -d
```

### 2. 端口 8080 被占用怎么办

现象：

- nginx 启动失败。
- 日志里出现端口绑定失败。

处理：

1. 先看有没有旧容器在跑：

```powershell
docker compose -f docker-compose.local-demo.yml ps
```

2. 如果是本项目旧容器，先停止：

```powershell
docker compose -f docker-compose.local-demo.yml down
```

3. 再启动：

```powershell
docker compose -f docker-compose.local-demo.yml --env-file .env.local-demo up -d
```

如果 8080 被其他软件占用，需要关闭那个软件，或之后再考虑调整 `.env.local-demo` 里的本地端口配置。

### 3. 页面打不开怎么办

先检查容器：

```powershell
docker compose -f docker-compose.local-demo.yml ps
```

再检查 nginx 和 api 日志：

```powershell
docker compose -f docker-compose.local-demo.yml logs --tail=100 nginx
docker compose -f docker-compose.local-demo.yml logs --tail=100 api
```

再试健康检查：

```powershell
Invoke-WebRequest -UseBasicParsing http://localhost:8080/api/healthz
```

### 4. healthz 不通怎么办

先看 `api` 是否 healthy：

```powershell
docker compose -f docker-compose.local-demo.yml ps
```

查看 api 日志：

```powershell
docker compose -f docker-compose.local-demo.yml logs --tail=200 api
```

常见原因：

- `.env.local-demo` 缺失或配置不完整。
- 数据库迁移失败。
- Docker Desktop 资源不足。
- 容器刚启动，还没 healthy，等十几秒再试。

### 5. worker 没起来怎么办

检查状态：

```powershell
docker compose -f docker-compose.local-demo.yml ps
```

查看 worker 日志：

```powershell
docker compose -f docker-compose.local-demo.yml logs --tail=200 worker-analysis
docker compose -f docker-compose.local-demo.yml logs --tail=200 worker-render
```

如果 Redis 不健康，worker 会无法正常接任务。先看 Redis：

```powershell
docker compose -f docker-compose.local-demo.yml logs --tail=100 redis
```

### 6. 生成报告失败时先看哪些日志

分析阶段失败，看：

```powershell
docker compose -f docker-compose.local-demo.yml logs --tail=300 worker-analysis
docker compose -f docker-compose.local-demo.yml logs --tail=300 api
```

渲染阶段失败，看：

```powershell
docker compose -f docker-compose.local-demo.yml logs --tail=300 worker-render
docker compose -f docker-compose.local-demo.yml logs --tail=300 api
```

重点搜索这些事件：

- `analysis.chart_started`
- `analysis.chart_completed`
- `analysis.chart_failed`
- `analysis.chart_partial_success`
- `job.awaiting_selection`
- `job.analysis_failed`
- `job.render_started`
- `job.render_completed`
- `job.render_failed`

### 7. 不要上传太大文件

本地体验版上传限制较小，建议使用小型 `csv/xls/xlsx` 文件。不要上传隐私数据、公司数据、身份证、手机号等敏感信息。

### 8. 本地体验版一次只建议跑一个任务

本地 demo 面向体验和演示，不是正式服务器。一次只建议跑一个分析任务，避免电脑卡顿或模型调用排队。
