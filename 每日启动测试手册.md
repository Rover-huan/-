## 一、每天最短启动流程

建议打开 **3 个 PowerShell 终端**。

### 第 1 步：启动 Redis

在任意终端输入：

```powershell
docker start smartanalyst-redis
```

可选检查：

```powershell
docker ps
```

你应该能看到容器名：

```powershell
smartanalyst-redis
```

如果提示 Docker 没启动，先打开 `Docker Desktop`，等它完全启动后再执行上面的命令。

---

### 第 2 步：启动后端

在第 1 个终端输入：

```powershell
cd C:\Users\lenovo\Desktop\实验\SmartAnalyst
powershell -ExecutionPolicy Bypass -File .\scripts\start-local-stack.ps1
```

正常情况下会看到类似信息：

```powershell
API PID: ...
Worker PID: ...
Beat PID: ...
Health check passed: http://127.0.0.1:8000/healthz
```

这表示：

- FastAPI 已启动
- Celery Worker 已启动
- Celery Beat 已启动
- 后端健康检查通过

后端地址固定看这个：

```powershell
http://127.0.0.1:8000
```

接口文档地址：

```powershell
http://127.0.0.1:8000/docs
```

---

### 第 3 步：启动前端

在第 2 个终端输入：

```powershell
cd C:\Users\lenovo\Desktop\实验\autoreport-ai-智能报告工作台
npm run dev
```

正常情况下会看到类似：

```powershell
Local: http://localhost:3000/
```

或者：

```powershell
Port 3000 is in use, trying another one...
Local: http://localhost:3001/
```

重点：

- 如果是 `3000`，就打开 `http://localhost:3000`
- 如果是 `3001`，就打开 `http://localhost:3001`

现在后端已经放行了 `3000` 和 `3001`，所以这两个端口都可以正常联调。

---

### 第 4 步：进入网页测试

在浏览器打开前端终端里显示的 `Local` 地址。

进入网页后，按这个顺序测：

1. 注册账号
2. 登录
3. 上传 `csv/xls/xlsx`
4. 等分析完成
5. 勾选候选图
6. 生成报告
7. 下载文件

## 四、如果网页打不开，先看这几个地方

### 情况 1：前端页面能打开，但登录时报 `Failed to fetch`

先检查后端是不是启动成功。

在终端输入：

```powershell
Invoke-WebRequest -UseBasicParsing http://127.0.0.1:8000/healthz
```

如果返回 `200`，说明后端是活的。

然后看前端终端显示的是 `3000` 还是 `3001`，浏览器要打开和它一致的地址。

---

### 情况 2：后端脚本启动失败

先检查 Redis：

```powershell
docker ps
```

如果没有 `smartanalyst-redis`，就重新启动：

```powershell
docker start smartanalyst-redis
```

然后重新执行：

```powershell
cd C:\Users\lenovo\Desktop\实验\SmartAnalyst
powershell -ExecutionPolicy Bypass -File .\scripts\start-local-stack.ps1
```

---

### 情况 3：前端提示 `Port 3000 is in use`

这不是错误，只是前端自动改用别的端口。

你只要打开终端里显示的 `Local` 地址即可，通常会变成：

```powershell
http://localhost:3001
```

---

### 情况 4：不能新建任务，提示活跃任务过多

到网页左侧的：

- `Active Jobs`
- `Recent Jobs`

里删除旧任务。

说明：

- 排队中 / 等待选图 / 等待渲染 / 已完成 / 失败 / 已过期：可以删除
- 正在分析 / 正在渲染：当前版本暂不支持强制中断

## 七、第一次之外，一般不需要重复做的事

下面这些不是每天都要做：

- 不需要每天重新安装依赖
- 不需要每天重新改 `.env`
- 不需要每天重新建 Redis 容器

只有在环境变动或报依赖错误时，才需要重新安装：

```powershell
cd C:\Users\lenovo\Desktop\实验\SmartAnalyst
python -m pip install -r requirements.txt
```

## 八、每天收工前你可以怎么做

如果你准备关机：

- 前端终端可以直接关
- 后端终端可以直接关
- Docker 可以不管，下次开机再 `docker start smartanalyst-redis`

如果你只是暂时离开，不关机，也可以保留它们继续运行。
